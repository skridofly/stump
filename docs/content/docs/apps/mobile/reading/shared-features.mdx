## Volumed-Based Navigation

<Callout type="warning">
    The experience of this feature on iOS is less than ideal due to limitations in the platform's handling of volume button events. The API surfaces are not overly ergonomic, as they are on Android
</Callout>

You may opt-in to use volume-based navigation in some readers, which allows you to navigate through the book using the device's physical volume buttons (e.g., "volume up" making the reader progress forward).
